Two layers of "on"

Master and sync mode are deliberately independent: keep the rest of the spoon's affordances even when auto-syncing is off. Each in-pane sublabel covers its specific scope.

Why two display sections?

"Detected Displays" answers what's plugged in right now — pure reference, regenerated on open. "Group Membership" is the editable assignment. UUID lives in the inventory only, where it disambiguates identical models.

Fixed pool of 10 groups

All ten letters A–J always exist; users just decide which to populate. No creation/deletion lifecycle. See vocabulary.md "Sync group → Why 10" for rationale.

Letter is identity, label is display

Letter is the stable ID; label is optional display sugar. See vocabulary.md "Sync group" + "Group label".

The single-group constraint

Each row's selector picks one group or None. The data model is a groupOf map keyed by position, so overlap is impossible by construction — not a runtime check.

Stale entries with memory

Configured-but-disconnected positions surface in the row stream with their last-seen timeline and prior group letter pulled from the _lastSeen map. Replug and the entry rejoins.

Environment-precondition banner

The red banner at the top fires when "Displays have separate Spaces" is OFF — surfacing the #1 install-time gotcha in the GUI rather than the Hammerspoon console. Hidden when env checks pass.

Two surfaces, one path

Every GUI edit autosaves to ~/.hammerspoon/SpacesSync.json. Hand-editing the JSON in any text editor triggers the same code path via hs.pathwatcher. Both surfaces converge on the spoon's apply routine — there is no Apply button.

Tooltips, not docs

Each timing field has a hover-info tooltip that explains the why, not just the what. The numbers exist to compensate for specific macOS behaviors — the user shouldn't have to read source comments to know why 0.3s and not 0.1s.

Why Sync Now shows two shortcuts

Sync Now keeps its inline kbd hint on the control panel as a discoverability cue; the Hotkeys section is where the binding is actually changed. The duplication is intentional.
